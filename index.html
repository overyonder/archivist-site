NAS Historian technical details

01 / CACHE

Binary cache format

The cache records Vault file identities and digests. Later scans use this index instead of reading unchanged file contents again.

02 / ZFS

ZFS change detection

Historian can compare ZFS snapshots to identify which paths changed between scans.

03 / STATX

Linux identity checks

On other Linux filesystems, statx metadata and inode identity allow existing files and links to reuse recorded digests.

04 / DIGESTS

Filesystem digest support

Historian can use fs-verity digests where available. It can also store computed digests in extended attributes so later scans do not need to hash the file again.
