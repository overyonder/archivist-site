    NAS Historian: storage-native Rust

    One static appliance binary. Historian is compiled as a musl static PIE for broad x86-64 Linux appliance compatibility, without a glibc, Electron, or .NET runtime in the storage path.

    Compute beside the disks. The NAS inventories files, hashes logical ROM contents and matches selected DAT evidence locally. Authenticated WebSockets carry live status; bulk snapshots use a gzip envelope with MessagePack metadata and a typed columnar entry table.

    Resident evidence, not raw DATs. Static policy reduced 1,221 selected manifests to 292 contributing sources, 313,914 releases and 1,239,634 unique strongest digests. The 50 MiB compiled catalogue memory-maps and hydrates warm in 0.557 seconds.

    Restartable hashing. Per-file logical digests live in a compact SQLite WAL store. Size + nanosecond mtime validation reuses completed records after restart; two contiguous workers reuse 1 MiB buffers and avoid HDD seek thrash.

    Live while you work. Filesystem notifications publish immutable generations; same-filesystem moves use a guarded server-side rename rather than copying ROM bytes over the desktop.

    Built for filesystem-native evidence. The next providers use ZFS snapshot differences to find changes in one sweep, statx identity to reuse hard-link digests, fs-verity for kernel-enforced hashes, and xattrs so Archivist digests can remain attached to the files themselves.

Measured implementation

Historian technical details

01 / deployment

Static musl appliance binary

A stripped x86-64 static PIE removes the glibc dependency and ships the native Rust storage service as one file. It is designed for Linux NAS appliances such as TrueNAS SCALE; architecture and kernel compatibility still apply.

02 / data locality

Hash where the bytes live

The desktop never pulls whole ROMs across SMB or NFS to identify them. Historian reads locally, computes CRC32, MD5 and SHA-1 in one pass, verifies logical ZIP members, and performs guarded server-side moves.

03 / resident catalogue

Selected evidence compiler

Static policy and overrides prune discarded branches before runtime. The measured artefact retains 292 of 1,221 sources, 313,914 releases and 1,239,634 unique strongest digests in 50 MiB. Warm mmap + native hydration takes 0.557 seconds.

04 / hydration

222,242 entries applied in 420 ms

The 210,840-file, 11,402-directory tree is available while hashing continues. Its 4.70 MB compressed replica fetched in 51 ms, crossed Tauri IPC in another 28 ms, decoded into the frontend in 227 ms and was fully applied 420 ms after the request. A hot process reached the applied state in 1.167 seconds.

05 / live protocol

HTTP bulk, WebSocket propagation

Authenticated binary WebSockets carry status and generation notices. Bulk HTTP uses gzip once, then MessagePack metadata plus parent-indexed SoA columns: one UTF-8 basename decode, typed-array metadata views and linear path reconstruction. The two-million-record digest graph remains native.

06 / cache-aware native path

Binary digests, reused buffers

CRC32, MD5 and SHA-1 are computed together with assembly-backed digest crates. Fixed-width binary keys and Fx hash maps avoid hex parsing and SipHash; two contiguous workers reuse 1 MiB buffers. Status reads are O(1), and the encoded snapshot is cached rather than rebuilt per request.

07 / mutable history

SQLite WAL, shipped

Logical per-file digest records use SQLite WAL in NORMAL synchronous mode, batched 256 files at a time with passive checkpoints. Hashing happens outside the short write transaction. Progress ticks stay memory-only, while immutable UI replicas are atomically replaced.

08 / filesystem evidence

Designed beyond path + read

Planned providers include ZFS snapshot deltas, Linux statx identity for hard-link reuse, fs-verity digests and optional digest xattrs. Each provider must state exactly what its evidence proves.

09 / stable invalidation

Policy changes rebuild policy

The catalogue fingerprint covers selected manifests, policy tables and parser/selection code—not Cargo.lock or transport code. Codec and dependency changes therefore keep the sub-second resident artefact path instead of causing an unrelated multi-minute policy rebuild.

10 / restart behaviour

Fast UI, resumable evidence

The cached stat tree is explicitly a last-scan snapshot, not a freshness promise. The frontend displays its scan time. Watchers update live generations while Historian runs; WAL digest records make hashing restartable without sending raw ROM data or credentials into JavaScript.
